TabVacuum — Privacy Policy

Last updated: February 2026

Overview

TabVacuum is a Chrome extension that saves your open browser tabs into locally stored sessions. We are committed to protecting your privacy. This policy explains what data the extension accesses, how it is used, and how it is stored.

Data Collection

When you click the TabVacuum icon to "vacuum" your tabs, the extension collects the following data from your currently open tabs:

  • Tab metadata: Page title, URL, and favicon URL
  • Page content: Up to 2,000 characters of text extracted from the visible page content
  • Tab state: Whether the tab was pinned

This data is collected only when you explicitly trigger the vacuum action. The extension does not run in the background, does not monitor your browsing, and does not collect data automatically.

Data Storage

All collected data is stored locally on your device using IndexedDB (via Dexie.js). No data is transmitted to any external server owned or operated by TabVacuum's developers. We do not have servers, databases, or any infrastructure that receives your data.

AI Features (Optional)

TabVacuum offers optional AI-powered features including smart session naming, tab summarization, semantic search, smart grouping, and session chat. These features:

  • Are entirely optional and disabled by default
  • Require you to provide your own OpenAI API key
  • Send tab titles, URLs, and extracted page content directly to OpenAI's API
  • Use your API key stored in your browser's localStorage

When you use AI features, data is sent directly from your browser to OpenAI. We never see, intercept, or store this data. OpenAI's handling of this data is governed by OpenAI's Privacy Policy.

Data Sharing

We do not sell, trade, or transfer your data to any third parties. Data only leaves your device if you explicitly use the optional AI features, in which case it is sent directly to OpenAI via your own API key.

Data Retention and Deletion

Your data is stored locally until you delete it. You can delete individual tabs, entire sessions, or all data at any time through the TabVacuum dashboard. Uninstalling the extension removes all stored data.

Permissions

TabVacuum requests the following permissions:

  • tabs: To read and manage open tabs during the vacuum process
  • storage: To temporarily bridge data between the service worker and dashboard
  • activeTab: To access the current tab for content extraction
  • scripting: To inject a content script that extracts page text
  • host_permissions (all URLs): To extract content from any webpage during vacuum

Limited Use Disclosure

TabVacuum's use of information received from Chrome APIs adheres to the Chrome Web Store User Data Policy, including the Limited Use requirements.

Children's Privacy

TabVacuum does not knowingly collect data from children under 13. The extension is intended for general audiences.

Changes to This Policy

We may update this privacy policy from time to time. Changes will be reflected on this page with an updated revision date.
